    面向对象三大特性：
            封装：内部操作对外部不可见(保护机制);
            继承：在上一辈的基础上继续发展;
            多态：（三者的实现）1、重写父类方法 2、面向接口编程 3、获取类的多样性
    
    程序设计的过程（流程）：
    OOA:面向对象分析（需求分析）
    OOD:面向对象设计 （实现过程）
    OOP:面向对象编程 （实现）
    更多的考虑内存分配问题

第二课：
 
    类与对象
    1.类就是对对象共性（概念）特征的抽象。
    2.对象所有的行为都要在类中进行操作，否则无法使用。
    3.类中的方法就是对象的具体行为。
    4.类中的属性就是对象的具体特点。

第四课:(对象内存分配)

    引用类型指的就是内存空间操作。
    两块内存空间：
        1.堆内存空间    保存真正的数据，对象的内存属性信息。
        2.栈内存空间    堆内存的引用地址，堆内存的操作权

第五课:

    类与对象（引用传递）
    1.引用别名放在栈内存中。一个堆内存可以被多个栈内存引用。
    2. 垃圾内存：没有任何栈内存所引用的堆内存空间。
    GC(垃圾回收机制)会降低程序的性能。在开发过程中降低垃圾内存空间的产生。

第六课:

    private 实现封装处理
    1、private实现封装的最大特征在于：只允许本类访问，外部不可直接访问。
    2、这仅仅是封装的第一步。
